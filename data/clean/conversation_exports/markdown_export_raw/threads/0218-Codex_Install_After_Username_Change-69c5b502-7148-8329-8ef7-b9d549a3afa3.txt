[ASSISTANT]
You’ve basically broken Windows’ path assumptions by renaming the user folder + registry. Codex isn’t the problem—your environment is. Fix that first or installs will keep failing.

Here’s the straight path to get Codex working again:

---

## 1. Verify your user path is actually usable
Open CMD and run:
```
echo %USERPROFILE%
```

If it still shows the **old username path**, your registry edit didn’t fully take.

Check this key:
```
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList
```
Find your SID → make sure:
```
ProfileImagePath = C:\Users\YourNewName
```

If this is wrong → fix it and reboot.

---

## 2. Fix broken environment variables (this is what usually kills installs)
Go to:
```
System Properties → Advanced → Environment Variables
```

Fix these if they still point to old username:

- `USERPROFILE`
- `HOMEPATH`
- `HOMEDRIVE`
- `APPDATA`
- `LOCALAPPDATA`

They should look like:
```
C:\Users\YourNewName\...
```

If even one points to the old folder → installs break silently.

---

## 3. Clean Node/npm (Codex depends on this)
Codex CLI installs through npm citeturn0search3

Blow away broken cache tied to old path:
```
npm cache clean --force
```

Then check where global packages go:
```
npm config get prefix
```

If it shows old username → fix it:
```
npm config set prefix "C:\Users\YourNewName\AppData\Roaming\npm"
```

---

## 4. Reinstall Codex clean
Now install fresh:
```
npm install -g @openai/codex
```

Then run:
```
codex
```

---

## 5. If it still fails (common after manual rename)
You’ve got deeper Windows damage. Two solid fixes:

### Option A (quick fix)
Create a symlink from old path → new path:
```
mklink /D C:\Users\OldName C:\Users\NewName
```

### Option B (clean fix — recommended)
- Create a **new Windows user with correct name**
- Move files over
- Delete the broken profile

Because manual renaming user folders is notorious for breaking permissions and app installs.
