# 1. Troubleshooting Notes (Electrical / Interference)

**Issue:** TV screen blacks out when ignitor is triggered.

**Observed Behavior:**
- Momentary signal loss (~2 seconds)
- Only occurs during spark discharge
- Display recovers without user input

**Root Cause:**
High-voltage ignition discharge is generating broadband electromagnetic interference (EMI). The TV or its input signal (HDMI) is being disrupted, not damaged.

**Where it’s happening:**
- Spark gap → radiated EMI  
- HDMI cable acting as an antenna  
- TV input circuitry briefly losing sync  

**Key Point:**
This is not a power drop. It’s signal corruption.

**Mitigation Options:**
- Shield the ignition lead (braided grounding sheath)
- Increase distance between ignitor and display cabling
- Add ferrite cores to HDMI cable
- Ground the metal frame near ignition source

**Advanced Angle:**
Containment is easier than redirection. Capturing that energy for reuse is inefficient due to its chaotic discharge profile.

# 2. Technical Explanation (Phone Unlocking / Firmware Stack)

Carrier locking does not sit in the Android OS layer. It exists lower.

**Stack Breakdown:**
- Application Layer → irrelevant  
- Android OS → reads lock status  
- Radio Interface Layer (RIL) → communicates with modem  
- Baseband Firmware → enforces lock  
- Carrier Database → validates unlock eligibility  

**Unlock Methods:**

1. **Official Unlock Code**
   - Generated from IMEI
   - Verified by carrier database
   - Written to baseband secure storage

2. **3rd Party Services**
   - Not “hacking” the phone
   - They access carrier-side unlock authorization channels
   - Same end result: valid unlock flag written

3. **Firmware Flashing**
   - Does NOT remove lock unless baseband is altered
   - Most consumer tools don’t touch this layer

**Conclusion:**
The lock persists across factory resets because it is stored in the modem subsystem, not user storage.
