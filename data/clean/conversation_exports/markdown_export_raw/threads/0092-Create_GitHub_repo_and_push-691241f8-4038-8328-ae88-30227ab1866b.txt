## Option 1 — Fastest via GitHub + your local machine

### 1. Create the empty repo on GitHub

1. Go to GitHub → click **New** repository.
2. Name it: `ConfluenceChronicles-StoryBible`
3. **Important:** choose:
   - Visibility: whatever you want (probably **private** for now)
   - **DO NOT** initialize with README, .gitignore, or license (leave it empty)
4. Click **Create repository**.

On the final page you’ll see something like:

```bash
git remote add origin https://github.com/USERNAME/ConfluenceChronicles-StoryBible.git
git push -u origin main
```

Keep that page open.

### 2. Unzip and set up the repo locally

On your machine:

1. Put `ConfluenceChronicles-StoryBible-Final-Ultimate.zip` somewhere, e.g. `C:\projects\`.
2. Extract it so you end up with something like:

   `C:\projects\ConfluenceChronicles-StoryBible-Final-Ultimate\...`

3. Open **Git Bash** or terminal and run:

```bash
cd "C:/projects/ConfluenceChronicles-StoryBible-Final-Ultimate"

# Initialize git repo
git init

# (Optional but smart) rename main branch
git branch -M main

# Add everything
git add .

# First commit message can be whatever
git commit -m "Initial commit: ConfluenceChronicles Story Bible (Final Ultimate)"
```

### 3. Hook it to the GitHub repo and push

Now connect it to the repo you created:

```bash
git remote add origin https://github.com/YOUR_USERNAME/ConfluenceChronicles-StoryBible.git

git push -u origin main
```

Replace `YOUR_USERNAME` with your GitHub username.

That’s it — your zip contents are now the `main` branch of that repo.

- Replace `YOUR_USERNAME` and paths with real values.

### Option A – Force your local as the truth (nuclear, but clean)


- Make sure there’s **no `.git` folder** inside what you’re uploading.

- If GitHub whines about size:
  - Individual file hard limit is ~100MB. Your Story Bible files will be miles under that.
