## The only guaranteed RAM dump
### Restart

## Windows 10/11: free RAM without rebooting

### 1) Kill the actual hogs
1. Press **Ctrl + Shift + Esc** → **Task Manager**
2. Sort by **Memory**
3. End tasks you recognize and don’t need (browsers with 80 tabs, launchers, Adobe, game overlays, etc.)

If it’s “Service Host” or “Antimalware Service Executable,” don’t randomly kill it—fix the cause (see below).

---

### 2) Stop the “startup bloat” that re-inflates RAM every boot
Task Manager → **Startup** tab → disable stuff you don’t need running 24/7 (Discord, Teams, Steam, Epic, updaters, etc.)

---

### 3) Clear the Windows standby cache (safe, fast, sometimes a big win)
Windows uses “Standby” RAM as cache; it usually releases it automatically, but sometimes it gets sticky and you feel it.

**Option A (cleanest): Microsoft Sysinternals RAMMap**
- Download **RAMMap** (Sysinternals)
- Run as admin → **Empty** menu → **Empty Standby List**

This doesn’t delete files. It just flushes cached RAM.

**Option B (command-line, admin):**
Use Sysinternals `EmptyStandbyList.exe` (same idea, scriptable). If you want this route, tell me your Windows version and whether you prefer portable tools.

---

### 4) Reduce browser RAM carnage
Chrome/Edge:
- Close tabs you don’t need
- Remove extensions you don’t trust
- Turn on “Sleeping tabs” (Edge) or “Memory Saver” (Chrome)

Browsers are usually the #1 RAM leak you *choose*.

---

### 5) Make sure your pagefile isn’t sabotaging you
If RAM is tight, a disabled/tiny pagefile makes everything worse.
- Control Panel → System → Advanced system settings → Performance → Advanced → Virtual memory
- Set to **System managed size** (recommended)

---

## Fast diagnosis (so you don’t fight ghosts)
In Task Manager → **Performance → Memory**, look at:
- **In use**: actual RAM being used
- **Available**: what you can use immediately
- **Cached**: standby cache (often safe to ignore unless you’re hitting limits)
- If **Committed** is near limit, you need pagefile / less background junk / more RAM.
