        if part and fn:
            return f"fastboot flash {part} {fn}"

        if var:

    elif op == "oem":
        var = step.attrib.get("var")
        if var:
            return f"fastboot oem {var}"
    elif op == "erase":
        part = step.attrib.get("partition")
        if part:
            return f"fastboot erase {part}"
    elif op == "reboot":
        return "fastboot reboot"
    elif op == "reboot-bootloader":
        return "fastboot reboot-bootloader"
    elif op == "continue":
        # common in some scripts
        return "fastboot continue"

Those messages aren’t a BAT problem. Your device is **rejecting specific partitions**.

### What the warnings mean (the harmless stuff)

- **“skip copying … avb footer (partition size: 0 …)”** on `gpt` / `preloader`  
  This is a known fastboot quirk: the device reports partition size as 0 for some slots/partitions, so fastboot can’t append an AVB footer and prints a warning. Flashing can still succeed. citeturn0search2

- **“Did you mean to fastboot format this ext4 partition?”** during `fastboot erase nvdata`  
  Just fastboot being chatty. `erase` != `format`. Not your real failure.

### The real failure: **(bootloader) Preflash validation failed**
This is the bootloader saying: **“Nope, I’m not accepting that image for that partition.”** The important part is that it happens on:

- `vbmeta_a`
- `vbmeta_system_a`
- `init_boot_a`
- `vendor_boot_a`
- `super` (and it explicitly warns you that vbmeta+vbmeta_system must be flashed first)

Common reasons:

1) **You’re trying to flash an OLDER build than what the phone currently has (anti-rollback / downgrade block).**  
   This is a classic trigger for “preflash validation failed.” If the bootloader thinks the image is older than what’s already installed, it refuses. citeturn0search9

2) **Wrong mode for some partitions: you need *fastbootd* (userspace fastboot) for certain flashes (often `init_boot` / dynamic partitions like `super`).**  
   For some devices, “preflash validation failed” on `init_boot` goes away when flashing from fastbootd. citeturn0search4

3) **Locked / secure flashing restrictions (bootloader not unlocked or OEM blocks those partitions).**  
   If the device is locked, it may allow some flashes but refuse AVB-critical partitions like vbmeta/init_boot/vendor_boot/super.

from pathlib import Path

def add_failstop(in_path: Path, out_name: str):
    txt = in_path.read_text(encoding="utf-8", errors="replace")
    lines = [l.rstrip("\r\n") for l in txt.splitlines()]
    out = []
    saw_header = False
    for line in lines:
        if not saw_header and line.strip().lower().startswith("@echo"):
            saw_header = True
            out.append(line)
            # add a quick device check after header
            out.append(".\\fastboot devices || goto :fail")
            continue
        if line.strip().lower().startswith(".\\fastboot "):
            out.append(f"{line} || goto :fail")
        else:
            out.append(line)
    # ensure fail label exists
    if not any(l.strip().lower() == ":fail" for l in out):
        out += [
            "",
            "echo.",
            "echo [FAIL] Stopped on an error.",
            "pause",
            "exit /b 1",
            "",
            ":fail",
            "echo.",
            "echo [FAIL] Stopped on an error.",
            "pause",
            "exit /b 1",
        ]
    out_path = in_path.with_name(out_name)
