Short answer: memory is scoped, yeah—but there are *two* layers you’re seeing.

Here’s how it actually works:

1. **“Memory (this project)” / project default**  
   - This controls whether I remember things **inside that specific project’s chats**.  
   - Stuff I learn in that project (like repo details, story structure, etc.) is **only used within that project**.  
   - Other projects and regular one-off chats don’t automatically see that project memory.

2. **“Memory for chats outside this project”**  
   - This controls whether I can store and use memory in **normal chats and other projects**, separate from the current project.  
   - Turning this off means: I don’t update or use your global memory from outside this project.  
   - Turning it on means: I can still remember general long-term stuff about you across all non-project chats (and other projects, depending on how OpenAI wired it), but that’s still distinct from this project’s scoped memory.

So:

- **Yes**, the “project memory” itself is only for that project’s chats.  
- The “memory for outside chats” is your broader/global memory behavior, not limited to that project.
