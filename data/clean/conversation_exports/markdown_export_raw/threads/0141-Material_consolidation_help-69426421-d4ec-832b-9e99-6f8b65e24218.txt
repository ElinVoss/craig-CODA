### First: what LS24 can and can’t do
**LS24 is material-centric** (it shows which bins/quants a *given material* is sitting in). It will not give you the full-warehouse picture by itself. For “the whole warehouse and where things can/can’t go,” you need **bin-centric** exports too (mainly **LX03 + LX01**, and ideally bin master/config tables if you can).

## A) Full warehouse “map” + what’s occupied/empty (bin-centric)

### 1) LX03 — Bin Status Report (this is the backbone)
SAP describes LX03 as showing whether bins are occupied or empty, and showing quants when occupied. citeturn1search19

**Steps**
1. In SAP command field, type **/nLX03** and Enter.
2. Enter:
   - **Warehouse Number (LGNUM)** = your warehouse
   - **Storage Type (LGTYP)** = run **one storage type at a time** (repeat per type)
   - Leave **Storage Bin** blank if allowed; if not, put `*` or use the **Multiple Selection** icon and enter a range/pattern.
3. Execute (F8).
4. Export:
   - Menu **List → Export → Spreadsheet** citeturn1search8
   - Choose **XLSX** (or CSV), save as `LX03_<LGNUM>_<LGTYP>.xlsx`

**Result:** every bin in that storage type, and whether it’s empty/occupied.

---

### 2) LX01 — List of Empty Storage Bins (proves what’s available)
LX01 is the standard report for empty bins. citeturn1search3turn1search2

**Steps**
1. **/nLX01**
2. Enter **LGNUM** and **LGTYP** (repeat per storage type)
3. Execute (F8)
4. **List → Export → Spreadsheet** citeturn1search8  
   Save as `LX01_<LGNUM>_<LGTYP>.xlsx`

**Result:** clean list of where things *can go right now* (at least physically empty).

## B) What’s actually in the warehouse (stock detail)

### 3) LS24 — Bin locations for a specific material (spot-check + “why is it there”)

**Steps**
1. **/nLS24**
