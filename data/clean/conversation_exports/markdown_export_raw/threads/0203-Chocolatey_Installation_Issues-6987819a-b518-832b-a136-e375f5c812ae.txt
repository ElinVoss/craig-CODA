Two separate problems in that log:

## 1) Chocolatey “package not found” is just wrong package names
Your script tried to install packages that **don’t exist under those names**.

Use these instead:

```powershell
# ADB + Fastboot (platform-tools equivalent)
choco install adb -y

# Samsung USB driver package name (singular)
choco install samsung-usb-driver -y
```

Those packages exist on Chocolatey as **`adb`** and **`samsung-usb-driver`**. citeturn0search10turn0search2  
If you want the old-school full SDK package: **`android-sdk`** exists too. citeturn0search7

So: your script’s `android-sdk-platform-tools` and `samsung-usb-drivers` (plural) are just busted names.

---

## 2) Heimdall “Unexpected handshake response” = USB driver/interface mess (usually from Zadig)
This exact failure pattern shows up when Heimdall can “see” the device but is talking to the **wrong USB interface/driver**. citeturn1view0

Do this in order (don’t skip steps):

### A) Stop anything that grabs Samsung USB
- Close **Smart Switch / Kies** if installed.
- In Task Manager, kill anything Samsung-related that’s running.

### B) Switch the physical connection (this matters more than people want to admit)
- Use a **different cable** (data cable, not charge-only).
- Use a **rear USB-A port** on the PC if possible.
- Avoid hubs/extension cables.

### C) Clean the driver situation (you likely have multiple “samsung_android” entries)
1. Unplug the phone.
2. Open **Device Manager** → **View → Show hidden devices**.
3. Expand these sections and uninstall anything Samsung/USB-driver-ish that looks related:
   - **Universal Serial Bus devices**
   - **libusbK USB Devices** (if present)
   - **Ports (COM & LPT)**
   - **Modems**
4. For each uninstall: check **“Delete the driver software for this device”** when offered.
5. Reboot the PC.

### D) Reinstall drivers the sane way, then apply Zadig to the correct interface
1. Install Samsung driver (optional for Heimdall, but good baseline):
   ```powershell
   choco install samsung-usb-driver -y
   ```
2. Boot phone to **Download Mode** and plug it in.
3. Run **Zadig as Administrator**:
   - **Options → List All Devices**
   - In the dropdown, pick **the Download-Mode device** (commonly **“Samsung USB Composite Device”** or **“Gadget Serial”**)
   - **CRITICAL:** if you see multiple entries like *Interface 0 / Interface 1 / Interface 2* — pick **Interface 0**.
   - Driver to install: start with **WinUSB**. If that still fails, repeat and try **libusbK**.
   - Click **Replace Driver**.

### E) Test the link before flashing
Run this first:

```powershell
heimdall print-pit --verbose
```

- If **that** still throws handshake/protocol init errors, flashing will never work until the USB interface/driver/cable/port is fixed.

If `print-pit` succeeds, then flash:

```powershell
heimdall flash --RECOVERY recovery.img --no-reboot --verbose
```
